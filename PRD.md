# PRD: KIS 투자관리 대시보드

## 1. 프로젝트 개요

### What
한국투자증권(KIS) OpenAPI 기반 개인 자산관리 대시보드.
다수의 증권 계좌와 수동 자산(부동산, 타 증권사, 예금 등)을 통합하여
총 자산 현황, 자산 배분, 수익률, 시장 인사이트, 투자 저널을
하나의 화면에서 관리한다.

### Why
- 여러 증권 계좌에 분산된 자산을 하나로 통합 조회할 수 없음
- 입출금이 빈번한 계좌에서 단순 손익은 실제 투자 성과를 반영하지 못함
- ETF 기반 자산배분을 주식/채권/금/국가/통화별로 다각도 분석할 수단이 없음
- 매수/매도 판단 근거가 기록되지 않아 동일 실수를 반복함
- 보유 종목과 시장 뉴스/분석의 연결고리가 없음

### Who
- 다수의 KIS 증권 계좌를 보유한 개인 투자자 (1인 사용)
- 주식, 국내채권, 해외채권, ETF, ELS, RP, 발행어음 등 다양한 상품 투자
- KIS API 미지원 자산(타 증권사, 부동산, IRP 등)도 수동 등록하여 통합 관리

## 2. 핵심 문제 정의

| 문제 | 설명 |
|------|------|
| **수익률 왜곡** | 급여 등 노동소득 유입과 투자 수익을 구분하지 못함. 자산 증가의 원인(투자수익 vs 신규 유입)을 분리해야 정확한 성과 측정 가능 |
| **자산 배분 오류** | 주식 계좌의 채권ETF를 '주식'으로, 중국ETF를 'KRW'로 잘못 분류하는 등 ETF 특성을 무시한 단순 분류가 왜곡된 배분 현황을 보여줌 |
| **자산 파편화** | KIS API가 미지원하는 자산(해외채권, ELS, RP, 발행어음, 타 증권사)이 포트폴리오에서 누락되어 전체 그림이 불완전 |
| **투자 경험 휘발** | 매수/매도 시점의 판단 근거가 기록되지 않아 동일 실수 반복. 과거 판단의 결과를 추적할 수단 없음 |
| **시장 정보 단절** | 내 포트폴리오와 시장 뉴스/분석의 연결고리 부재. 다양한 관점(상승론/하락론/가치/성장)의 균형 잡힌 인사이트 필요 |

## 3. 메뉴 구조 (6개)

### 3.1 대시보드
전체 자산을 한눈에 파악하는 메인 화면.
- 총 자산 금액 (KIS + 수동자산 합산, 원화 환산)
- 전일/전주 대비 손익 (금액 + 수익률)
- 자산군별 비중 요약 차트 (주식/채권/현금/금/대체투자)
- 상위 보유 종목 리스트
- 포트폴리오 가치 추이 미니 차트
- 최근 배당/이자 수입 요약

### 3.2 계좌별 현황
계좌 단위로 보유 자산을 상세히 확인하고 수동 자산을 관리하는 화면.
- 계좌별 아코디언: 국내주식, 해외주식, 채권, ELS, RP, 발행어음, 현금(원화/외화)
- 종목별 상세 모달 (매입가, 평가금, 수익률, 분류 태그 편집)
- 수동 자산 CRUD (타 증권사, 부동산, IRP, 예금, 대출 등)
- 과거 날짜 기준 조회 (타임머신)

### 3.3 자산 배분
자산배분 이론 관점에서 포트폴리오를 다각도로 분석하는 화면.
- **자산군별**: 주식 / 채권 / 현금 / 금·원자재 / 대체투자(암호화폐 ETF 등)
- **국가별**: 한국 / 미국 / 중국 / 인도 / 유럽 / 아시아 / 이머징 / 글로벌
- **통화별**: KRW / USD / CNY / EUR / 기타 — 헤지(H) 상품은 KRW로 분류
- **주식 구조**: Broad ETF / Factor / Sector / Theme / 개별주
- **채권 구조**: 만기별 래더, 표면이율 vs 실질이율(YTM), HTM vs MTM
- 각 섹션별 AI 인사이트 코멘트
- 인터랙티브 차트 (클릭 → 구성종목 팝업)

**ETF 분류 핵심 규칙**:
- ETF 이름 파싱으로 자산군/국가/통화/섹터 자동 분류
- 채권 ETF → 자산군 '채권', 금 ETF → '금', 비트코인 ETF → '대체투자'
- (H), (합성) 표기 → 통화 KRW (원화 헤지)
- 예: "차이나항셍테크ETF" → 국가: 중국, 통화: CNY

### 3.4 수익률 분석
주간 스냅샷 기반 수익률 추적과 현금흐름/인컴을 통합한 화면.

- **수익률 섹션**
  - 미실현 손익 현황 (종목별 + 전체)
  - 시계열 수익률 차트 (1W/1M/3M/6M/YTD/1Y/ALL)
  - TWR 기반 순수 투자 수익률 (현금흐름 보정)
  - 벤치마크 비교 (KOSPI, S&P500)
- **현금흐름 섹션**
  - 월별 순입금/순출금 추적 (노동소득 유입 vs 투자수익 구분)
  - "매달 얼마나 투자 자산에 추가하고 있는가" 지표
  - 자산 증가 원인 분해 차트 (투자수익 + 신규유입 + 인컴)
- **인컴 섹션**
  - 배당금 / 분배금 / 채권이자 내역
  - 연도별 인컴 합계 및 월평균
  - 종목별 배당/이자 이력

**스냅샷 전략**:
- 정기: 매주 금요일 16:00 KST (한국 장 마감 후)
- 수동: 사용자가 원할 때 즉시 스냅샷 (대량 매매, 시장 급변 등)
- 주간 스냅샷 시 해당 주의 입출금/배당/이자도 함께 정리

### 3.5 마켓 인사이트
내 포트폴리오와 시장 정보를 연결하여 투자 아이디어를 제공하는 화면.
- 보유 종목 관련 뉴스 자동 수집
- YouTube/블로그/애널리스트 리포트 요약 (AI)
- AI 포트폴리오 종합 코멘트 (배분 진단 + 시장 상황 기반 제안)
- 관심 종목 워치리스트

**정보 소스 관리**:
- 팔로우할 소스 리스트를 별도 파일로 관리 (유튜브, 블로그, 뉴스레터 등)
- 에이전트가 초기 추천, 월별 갱신 제안
- 다양한 관점 균형: 상승론자/하락론자, 가치/성장, 주식/채권, 국내/글로벌

### 3.6 투자 저널
매매 시점의 판단 근거를 기록하고, 사후 결과를 분석하는 화면.
- 매매 기록 + 사유 입력 (진입가, 목표가, 손절가)
- 보유 중 종목 메모/분석 작성
- 특정 날짜 이슈 기록 (예: FOMC, 실적발표)
- 과거 저널 검색 및 태그 필터
- 거래 자동 감지 → 저널 작성 프롬프트
- 결과 분석: 당시 판단 vs 실제 결과 비교

## 6. 제약사항 및 의존성

### KIS API
- **Rate Limit**: 실전 ~20회/초, 모의 ~2회/초, 토큰 발급 1회/분
- **확인 필요**: CMA 계좌 내 해외채권, RP, 발행어음의 API 조회 가능 여부
- **미지원 시 대체**: `manual_assets` 테이블로 수동 입력
- **계좌 타입별 차이**: 연금계좌(22, 29)는 특수 파라미터 필요
- **채권 조회**: 페이지당 최대 20건 (페이지네이션 필요)
- **해외주식**: 거래소별(NASD/NYSE/AMEX) 개별 호출
- **전체 API 카탈로그**: `docs/kis_api_catalog.md` 참조 (350+ API)

### 활용 예정 KIS API (주요)
- **배당/인컴 자동감지**: KSD 배당일정, 국내/해외 권리내역
- **수익률 분석**: 기간손익 일별합산, 기간별 매매손익
- **자산 종합 조회**: 투자계좌 자산현황 (주식+채권+펀드+현금 통합)
- **마켓인사이트**: 투자의견, 추정실적, 뉴스 API

### 데이터 의존성
- 환율: 한국수출입은행 API (평일만 제공, 주말/공휴일은 직전 영업일 fallback)
- AI 인사이트: OpenAI API 호출 비용 발생

### 사용자 환경
- 주 사용 환경: PC 브라우저 (모바일은 보조)
- 1인 사용 앱 (멀티유저 불필요)

## 7. 에이전트 설계 방향

이 프로젝트는 하네스 엔지니어링으로 개발하며, 다음 에이전트를 구상한다:

| 에이전트 | 역할 |
|---------|------|
| **내부 자산 관리** | 자산배분 전략 설계, ETF/채권 분류 규칙 관리, 리밸런싱 제안, 목표 vs 현재 갭 분석 |
| **외부 마켓 분석** | 팔로우 소스 큐레이션, 뉴스/콘텐츠 요약, 포트폴리오 연관성 분석, 다양한 관점 균형 유지 |
| **보안** | 커밋 전 시크릿 스캔, 코드 리뷰 시 보안 체크리스트, 환경변수/하드코딩 패턴 탐지 |

> 이 PRD와 `feature_list.md`는 하네스 초기 가이드라인이다.
> 에이전트가 구현 과정에서 더 좋은 구조나 기능을 발견하면 자유롭게 개선할 수 있다.
